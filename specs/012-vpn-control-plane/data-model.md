# Data Model: VPN Control Plane MVP

## VPNNetwork

Represents the single default virtual network managed by EchoCenter in Phase 1.
The model keeps an identifier so later phases can support multiple VPN networks
without rewriting peer ownership.

### Fields

- `id`: stable numeric identifier.
- `name`: display name, default `EchoCenter VPN`.
- `interface_name`: configured interface label, default `wg-echo`.
- `network_cidr`: VPN network CIDR, default `10.88.0.0/16`.
- `server_vpn_ip`: EchoCenter VPN IP, default `10.88.0.1`.
- `listen_port`: public WireGuard listen port, default `51820`.
- `domain`: VPN hostname, default `echocenter.vpn`.
- `enabled`: whether VPN control-plane behavior is enabled.
- `server_public_key`: WireGuard-compatible server public key.
- `encrypted_server_private_key`: protected server private key material.
- `created_at`: creation timestamp.
- `updated_at`: last update timestamp.

### Validation Rules

- `name` must be non-empty after trimming.
- `interface_name` must be non-empty and must not contain whitespace.
- `network_cidr` must parse as a CIDR.
- `server_vpn_ip` must parse as an IP address inside `network_cidr`.
- `listen_port` must be within `1..65535`.
- `domain` must be non-empty and suitable for hosts-file guidance.
- `server_public_key` must be a valid WireGuard-compatible public key when set.
- `encrypted_server_private_key` must never be exposed through API responses.

### State Notes

- Phase 1 manages only one default network record.
- If env settings change, startup/bootstrap reconciles the default record while
  preserving existing protected server key material.
- Disabling the network does not delete peers.

## VPNPeer

Represents one managed VPN peer for an agent, user, node, or external operator.

### Fields

- `id`: stable numeric identifier.
- `network_id`: parent `VPNNetwork` identifier.
- `name`: administrator-facing peer name.
- `agent_id`: optional EchoCenter agent/user identifier for an agent binding.
- `user_id`: optional EchoCenter user identifier for an owner binding.
- `public_key`: peer public key.
- `encrypted_private_key`: protected peer private key when EchoCenter generated
  the key pair; empty for externally generated keys.
- `private_key_available`: derived API flag indicating whether a complete client
  config can be generated by EchoCenter.
- `virtual_ip`: assigned single-host VPN address, such as `10.88.1.10`.
- `address_cidr`: peer address with prefix, such as `10.88.1.10/32`.
- `allowed_ips`: route list for the peer, default `<server_vpn_ip>/32`.
- `persistent_keepalive`: keepalive seconds, default `25`.
- `enabled`: peer administrative state.
- `last_handshake`: nullable or unknown until runtime integration.
- `rx_bytes`: nullable or unknown until runtime integration.
- `tx_bytes`: nullable or unknown until runtime integration.
- `created_at`: creation timestamp.
- `updated_at`: last update timestamp.

### Validation Rules

- `network_id` must reference the default VPN network.
- `name` must be non-empty and unique within a network.
- `public_key` must be valid and unique within a network.
- `agent_id` and `user_id` are optional, but if present they must reference
  existing EchoCenter users.
- `virtual_ip` must parse as an IP inside the parent network CIDR and must not
  equal the server VPN IP.
- `virtual_ip` must be unique within a network, including disabled peers.
- `allowed_ips` defaults to the server VPN IP as a single-host route.
- `persistent_keepalive` defaults to `25` and must be positive when supplied.
- `encrypted_private_key` must never be exposed outside generated client config
  for the matching server-generated peer.

### State Transitions

```text
created(enabled=true)
  ├── disable -> enabled=false
  ├── update profile fields -> enabled unchanged
  └── delete -> removed from active API views

enabled=false
  ├── enable -> enabled=true
  ├── update profile fields -> enabled unchanged
  └── delete -> removed from active API views
```

### Runtime Status Semantics

- `last_handshake`, `rx_bytes`, and `tx_bytes` are returned as unknown or empty
  in Phase 1.
- Phase 2 may populate these fields from live runtime data without changing the
  peer identity or config generation contract.

## PeerConfiguration

Represents generated setup material for a peer.

### Fields

- `peer_id`: peer identifier.
- `mode`: `full_client_config` when EchoCenter generated the peer private key,
  otherwise `server_fragment`.
- `wireguard_config`: full client config or server-side fragment text.
- `agent_websocket_url`: WebSocket URL using the VPN domain, for example
  `ws://echocenter.vpn:8080/api/ws`.
- `hosts_entry`: hosts-file guidance, for example `10.88.0.1 echocenter.vpn`.
- `warnings`: human-readable setup notes, including disabled peer state or
  missing peer private key for externally generated peers.

### Validation Rules

- Full client config may include `PrivateKey` only for the requested peer when
  EchoCenter generated and can decrypt that peer private key.
- Server private key is never included.
- Config generation fails for deleted peers.
- Config generation for disabled peers may return material only with a warning
  that the peer is disabled.

## Persistence

### `vpn_networks`

- `id` primary key.
- `name`, `interface_name`, `network_cidr`, `server_vpn_ip`, `listen_port`,
  `domain`, `enabled`.
- `server_public_key`, `encrypted_server_private_key`.
- `created_at`, `updated_at`.

### `vpn_peers`

- `id` primary key.
- `network_id` foreign key to `vpn_networks`.
- `name`, `agent_id`, `user_id`, `public_key`, `encrypted_private_key`,
  `virtual_ip`, `allowed_ips`, `persistent_keepalive`, `enabled`.
- `last_handshake`, `rx_bytes`, `tx_bytes`.
- `created_at`, `updated_at`.
- Unique constraints on `(network_id, name)`, `(network_id, public_key)`, and
  `(network_id, virtual_ip)`.
